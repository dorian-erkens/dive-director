# Product Requirements Documents

Ce dossier contient les PRDs du projet Dive Director, generes automatiquement a partir des [insights](https://github.com/dorian-erkens/dive-director/issues?q=label%3Ainsight).

## Workflow

```
1. Creer une issue avec le template "Insight"
2. GitHub Action genere un PRD draft via Claude API
3. Une PR est creee avec le PRD dans docs/prd/
4. Review, iteration, merge
```

## Framework

Chaque PRD suit le framework **GIST** (Goals, Ideas, Steps, Tasks) avec :
- **ICE Scoring** (Impact x Confidence x Ease)
- **Confidence Meter** d'Itamar Gilad (evidence-based scoring)
- **Hypotheses testables** pour du prototypage rapide
